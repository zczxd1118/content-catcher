## ✍️ 推荐写法

### 1. 一句话总结

用一句中文先把全篇钉死，方便读者决定要不要往下看。

### 2. 三到五个主题章节

每章 200–600 字，**不是字幕翻译**，是经过你（Claude）整理后的"观点 + 上下文 + 关键数据"。

## 🚫 不要做的事

- ❌ 不要直接复制 Whisper 转写或 RSS show notes（那是投喂包，不是文章）
- ❌ 不要保留"任务说明""请基于下方原文..."（那是给 LLM 看的）
- ❌ 不要在每段开头加"作者说道:""他提到:"（让观点直接出场）
- ❌ 不要堆叠时间戳（[00:12:34] ... [00:18:45] ...），换成主题章节
- ❌ 不要保留主播口头禅（"我是XX""欢迎评论区""下期见"）

## 🛠 这份文件该叫什么名字

放在对应 digest 目录下，命名为：

```
output/digest/<本期目录>/chapter-<视频标题或缩写>.md
```

例：

```
output/digest/weekly-2026-W23-20260602/chapter-Daytona-Latent-Space.md
output/digest/weekly-2026-W23-20260602/chapter-Vibe-Coding-小白教程.md
```

写完后跑：

```bash
catch.py --subscribe channels.yaml --finalize-epub weekly-2026-W23-20260602
```

EPUB 会用这些 `chapter-*.md` 当章节，把上面那些"什么是 chapter 模板"全部当成历史遗物——
你最终交付给读者的就是一份干干净净的优美长文集。
